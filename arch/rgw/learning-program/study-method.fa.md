# روش مطالعه عمیق هر فایل

برای **هر فایل** در برنامه یادگیری این پنج قدم را تکرار کن.

## قدم A — کشف نقش (۱۵ دقیقه)

1. این فایل در زنجیره HTTP کجاست؟ (frontend / process / rest / op / sal / rados)
2. کلاس‌های public کدام‌اند؟
3. **Find References:** چه کسی این کلاس را instantiate می‌کند؟

## قدم B — Happy path (۴۵–۹۰ دقیقه)

یک flow کامل (مثلاً GET) را از entry تا return دنبال کن. شاخه error را نادیده بگیر.

## قدم C — Failure path (۳۰ دقیقه)

یک خطای رایج (مثلاً bucket不存在) را از محل برگشت `-ENOENT` تا `abort_early` trace کن.

## قدم D — یادداشت (۱۵ دقیقه)

جدول کوچک:

| تابع | ورودی مهم | خروجی | side effect |
|------|-----------|--------|-------------|
| | | | |

## قدم E — لمس کد (اختیاری ولی قوی)

- یک `dout(20)` در نقطه‌ای که تازه فهمیدی
- یک درخواست HTTP و تطبیق لاگ

## زمان‌بندی پیشنهادی هر فاز

| فعالیت | ساعت |
|--------|------|
| خواندن هدفمند | ۳–۶ |

| خودارزیابی | ۰٫۵ |

## اشتباهات رایج

- خواندن فایل‌های چندهزار خطی از ابتدا بدون symbol
- پریدن به `RGWRados` قبل از `sal::`
- قاطی کردن `verify_requester` و `verify_permission`
